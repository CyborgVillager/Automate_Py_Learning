        <h1>Python - Error Types </h1>
        <p>
            The most common reason of an error in a Python program is when a certain statement is not in accordance with the prescribed usage. 
            Such an error is called a syntax error. The Python interpreter immediately reports it, usually along with the reason.
        </p>

        <samp>
            &gt;&gt;&gt; print "hello"<br />
            SyntaxError: Missing parentheses in call to 'print'. Did you mean print("hello")?
        </samp>
        <p>
            In Python 3.x, print is a built-in function and requires parentheses. The statement above violates this usage and hence syntax error is displayed.
        </p>


        <p>
            Many times though, a program results in an error after it is run even if it doesn't have any syntax error. Such an error is a runtime error, called an exception. 
            A number of built-in exceptions are defined in the Python library.
            Let's see some common error types.
        </p>

        <p>
            <strong>IndexError</strong> is thrown when trying to access an item at an invalid index.
        </p>

        <samp>
            &gt;&gt;&gt; L1=[1,2,3]<br />
            &gt;&gt;&gt; L1[3]<br />
            Traceback (most recent call last):<br />
            File "&lt;pyshell#18&gt;", line 1, in &lt;module&gt;
            <br />
            L1[3]<br />
            IndexError: list index out of range
        </samp>
        <p>
            <strong>ModuleNotFoundError</strong> is thrown when a module could not be found.
        </p>

        <samp>
            &gt;&gt;&gt; import notamodule<br />
            Traceback (most recent call last):<br />
            File "&lt;pyshell#10&gt;", line 1, in &lt;module&gt;
            <br />
            import notamodule<br />
            ModuleNotFoundError: No module named 'notamodule'
        </samp>
        <p>
            <strong>KeyError</strong> is thrown when a key is not found.
        </p>

        <samp>
            &gt;&gt;&gt; D1={'1':"aa", '2':"bb", '3':"cc"}<br />
            &gt;&gt;&gt; D1['4']<br />
            Traceback (most recent call last):<br />
            File "&lt;pyshell#15&gt;", line 1, in &lt;module&gt;

            <br />
            D1['4']<br />
            KeyError: '4'
        </samp>
        <p>
            <strong>ImportError</strong> is thrown when a specified function can not be found.
        </p>

        <samp>
            &gt;&gt;&gt; from math import cube<br />
            Traceback (most recent call last):<br />
            File "&lt;pyshell#16&gt;", line 1, in &lt;module&gt;
            <br />
            from math import cube<br />
            ImportError: cannot import name 'cube'
        </samp>
        <p>
            <strong>StopIteration</strong> is thrown when the <code>next()</code> function goes beyond the iterator items.
        </p>

        <samp>
            &gt;&gt;&gt; it=iter([1,2,3])<br />
            &gt;&gt;&gt; next(it)<br />
            1<br />
            &gt;&gt;&gt; next(it)<br />
            2<br />
            &gt;&gt;&gt; next(it)<br />
            3<br />
            &gt;&gt;&gt; next(it)<br />
            Traceback (most recent call last):<br />
            File "&lt;pyshell#23&gt;", line 1, in &lt;module&gt;
            <br />
            next(it)<br />
            StopIteration
        </samp>
        <p>
            <strong>TypeError</strong> is thrown when an operation or function is applied to an object of an inappropriate type.
        </p>

        <samp>
            &gt;&gt;&gt; '2'+2<br />
            Traceback (most recent call last):<br />
            File "&lt;pyshell#23&gt;", line 1, in &lt;module&gt;
            <br />
            '2'+2<br />
            TypeError: must be str, not int
        </samp>
        <p>
            <strong>ValueError</strong> is thrown when a function's argument is of an inappropriate type.
        </p>

        <samp>
            &gt;&gt;&gt; int('xyz')<br />
            Traceback (most recent call last):<br />
            File "&lt;pyshell#14&gt;", line 1, in &lt;module&gt;
            <br />
            int('xyz')<br />
            ValueError: invalid literal for int() with base 10: 'xyz'
        </samp>
        <p>
            <strong>NameError</strong> is thrown when an object could not be found.
        </p>

        <samp>
            &gt;&gt;&gt; age<br />
            Traceback (most recent call last):<br />
            File "&lt;pyshell#6&gt;", line 1, in &lt;module&gt;
            <br />
            age<br />
            NameError: name 'age' is not defined
        </samp>
        <p>
            <strong>ZeroDivisionError</strong> is thrown when the second operator in the division is zero.
        </p>

        <samp>
            &gt;&gt;&gt; x=100/0<br />
            Traceback (most recent call last):<br />
            File "&lt;pyshell#8&gt;", line 1, in &lt;module&gt;
            <br />
            x=100/0<br />
            ZeroDivisionError: division by zero
        </samp>
        <p>
            <strong>KeyboardInterrupt</strong> is thrown when the user hits the interrupt key (normally Control-C) during the execution of the program.
        </p>

        <samp>
            &gt;&gt;&gt; name=input('enter your name')<br />
            enter your name^c<br />
            Traceback (most recent call last):<br />
            File "&lt;pyshell#9&gt;", line 1, in &lt;module&gt;
            <br />
            name=input('enter your name')<br />
            KeyboardInterrupt
        </samp>

                            Raised when the assert statement fails.

                    <tr>
                        <td>
                            AttributeError
